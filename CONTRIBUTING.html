Contributing

Thank you for wanting to contribute. Add issues or PRs following the repository guidelines.

  • Open an issue describing the change.
  • Submit a PR with a branch per feature.
  • Maintain the repo style and add tests/documentation if applicable.

How to Contribute​

  1. Fork the Repository — Create your own copy of the project.
  2. Create a Feature Branch — Use descriptive names (e.g., feature/add-icons, fix/broken-links).
  3. Make Your Changes — Keep commits atomic and well-documented.
  4. Test Your Changes — Run npm run build to ensure everything works.
  5. Submit a Pull Request — Describe what you changed and why.

Code Style​

  • Use clear, descriptive commit messages.
  • Follow the existing file structure and naming conventions.
  • Keep documentation up-to-date when making changes.

Reporting Issues​

  • Use the GitHub Issues tab.
  • Provide a clear description of the problem.
  • Include steps to reproduce if applicable.
  • Attach screenshots or error logs when relevant.

Questions?​

Feel free to open a discussion or reach out via GitHub Issues.

Happy contributing!
